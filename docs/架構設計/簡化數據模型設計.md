# 簡化數據模型設計

## 設計原則
1. **簡單優先**：從9種房間狀態簡化為3種
2. **明確命名**：使用完整單詞而非縮寫
3. **必要屬性**：只保留核心業務屬性
4. **向後兼容**：設計遷移路徑

## 核心類型定義

### 房間狀態（從9種簡化為3種）
```typescript
// 簡化版房間狀態
export type SimpleRoomStatus = 
  | 'available'    // 空房可出租
  | 'occupied'     // 已出租入住中
  | 'maintenance'; // 維修中不可出租

// 狀態轉換規則：
// available → occupied (入住)
// occupied → available (退房)
// available ↔ maintenance (維修)
```

### 房間資料（從40+屬性簡化為15個核心屬性）
```typescript
export interface SimpleRoom {
  // 核心識別資訊
  id: string;           // UUID格式
  propertyId: string;   // 所屬物業ID
  number: string;       // 房號（如"101"）
  floor: number;        // 樓層
  
  // 租金資訊
  monthlyRent: number;  // 月租金
  deposit: number;      // 押金
  
  // 狀態資訊
  status: SimpleRoomStatus;
  
  // 租客資訊（僅在occupied狀態時有效）
  tenant?: {
    name: string;       // 租客姓名
    phone: string;      // 聯絡電話
  };
  
  // 租約資訊
  lease?: {
    checkInDate: string;    // 入住日期 YYYY-MM-DD
    checkOutDate: string;   // 到期日期 YYYY-MM-DD
  };
  
  // 電費資訊
  electricity?: {
    currentMeter: number;   // 本期電錶
    lastMeter: number;      // 上期電錶
    rate: number;          // 電費單價
  };
  
  // 時間戳記
  createdAt: string;
  updatedAt: string;
}
```

## 驗證規則

### 房間狀態驗證
```typescript
// 狀態轉換驗證
const validTransitions: Record<SimpleRoomStatus, SimpleRoomStatus[]> = {
  available: ['occupied', 'maintenance'],
  occupied: ['available'],
  maintenance: ['available']
};

function isValidTransition(from: SimpleRoomStatus, to: SimpleRoomStatus): boolean {
  return validTransitions[from].includes(to);
}
```

### 業務規則驗證
1. **入住規則**：
   - 必須有租客姓名和電話
   - 必須有入住日期和到期日期
   - 押金必須大於0

2. **退房規則**：
   - 必須結清所有未付款項
   - 必須記錄最後電錶讀數
   - 計算應退押金

## API設計要點

### 房間相關API
```
GET    /api/rooms           # 獲取房間列表
GET    /api/rooms/:id       # 獲取單個房間
POST   /api/rooms           # 創建房間
PUT    /api/rooms/:id       # 更新房間
DELETE /api/rooms/:id       # 刪除房間
POST   /api/rooms/:id/check-in   # 入住
POST   /api/rooms/:id/check-out  # 退房
POST   /api/rooms/:id/maintenance # 設為維修
```

### 付款相關API
```
GET    /api/payments        # 獲取付款記錄
POST   /api/payments        # 創建付款記錄
PUT    /api/payments/:id/status # 更新付款狀態
```
