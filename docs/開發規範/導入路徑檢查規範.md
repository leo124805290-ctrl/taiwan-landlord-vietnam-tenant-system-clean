# 導入路徑檢查規範

## 目的
避免因導入路徑錯誤導致的構建失敗和部署問題。

## 適用範圍
所有 TypeScript/JavaScript 導入語句。

## 規範內容

### 1. 路徑格式規範

#### 允許的路徑格式：
```typescript
// 1. 絕對路徑（使用別名）
import Component from '@/components/Component'
import { util } from '@/lib/utils'

// 2. 相對路徑（同一目錄或子目錄）
import Component from './Component'
import Util from './utils/Util'

// 3. 相對路徑（上一層目錄）
import Config from '../config'
```

#### 禁止的路徑格式：
```typescript
// ❌ 超過2層的相對路徑
import Component from '../../../components/Component'

// ❌ 不存在的別名
import Component from '@components/Component'  // 如果未配置

// ❌ 缺少路徑前綴
import Component from 'Component'
```

### 2. 別名配置規範

#### tsconfig.json 配置：
```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./*"],
      "@components/*": ["./src/components/*"],
      "@lib/*": ["./src/lib/*"],
      "@types/*": ["./src/types/*"]
    }
  }
}
```

#### 使用規則：
1. **一致性**：在整個專案中使用相同的別名
2. **明確性**：別名應該明確表示目錄類型
3. **簡潔性**：避免過多的別名配置

### 3. 文件移動和重構規範

#### 移動文件時：
1. **更新導入**：更新文件內的所有導入語句
2. **更新引用**：更新所有引用該文件的其他文件
3. **刪除舊文件**：確認無引用後刪除舊文件

#### 重構目錄時：
1. **批量更新**：使用腳本批量更新導入路徑
2. **逐步驗證**：每次更新後運行構建測試
3. **文檔更新**：更新相關文檔和示例

### 4. 檢查流程

#### 開發時檢查：
1. **編輯器檢查**：確保編輯器沒有顯示路徑錯誤
2. **TypeScript檢查**：運行 `npx tsc --noEmit`
3. **構建測試**：運行 `npm run build` 或 `next build`

#### 提交前檢查：
```bash
# 1. 檢查導入路徑
npx tsc --noEmit --skipLibCheck

# 2. 快速構建測試
npm run build --dry-run

# 3. 檢查未使用的導入
npx ts-unused-exports tsconfig.json
```

### 5. 常見錯誤和解決方案

#### 錯誤1：Cannot find module
**原因**：路徑不存在或文件已移動
**解決**：
1. 檢查文件是否存在
2. 檢查路徑是否正確
3. 更新導入語句

#### 錯誤2：Module not found
**原因**：別名未配置或配置錯誤
**解決**：
1. 檢查 tsconfig.json 配置
2. 確認別名是否正確
3. 使用相對路徑作為臨時解決方案

#### 錯誤3：Import trace errors
**原因**：鏈式導入錯誤
**解決**：
1. 從錯誤訊息的第一個文件開始檢查
2. 逐層檢查導入鏈
3. 確保所有中間文件路徑正確

### 7. 團隊協作規範

#### 代碼審查時：
1. **檢查導入**：審查所有新增的導入語句
2. **驗證路徑**：確認所有路徑都存在且正確
3. **檢查配置**：確認 tsconfig.json 配置正確

#### 合併代碼前：
1. **運行檢查**：必須通過導入路徑檢查
2. **構建測試**：必須通過構建測試
3. **文檔更新**：相關文檔必須更新

## 處罰條款

### 輕微違規：
- 導入路徑錯誤但未導致構建失敗
- **處罰**：警告並要求立即修正

### 嚴重違規：
- 導入路徑錯誤導致構建失敗
- 影響團隊其他成員開發
- **處罰**：暫停提交權限，直到問題解決

### 重複違規：
- 多次違反同一規範
- **處罰**：需要額外的代碼審查和培訓
