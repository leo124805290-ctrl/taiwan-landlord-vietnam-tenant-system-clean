### 2. 已知問題與解決方法

#### 問題 A：新增物業失敗（房間狀態錯誤）

**症狀：**
- 使用者「新增物業」時點下去會報錯「儲存房間失敗」
- Console 顯示 `res.status is not a function` / 500 錯誤

**原因：**
- 後端 API 腳本 `simple-api-fixed.js` 於第 603 行處理回應時，可能是 pipeline race condition
- API Server 正在處理大量請求時，response 物件狀態已損毀

**已採取的修補（預期正式生效位置）：**
```javascript
// lib/usage 修復文檔（已存在於代碼 log）
async function sendResponse(res, statusCode, data, success) {
  if (res.statusCode === undefined && res.headers === undefined) {
    console.warn('Response 物件無效');
    return;
  }

  // 如果已有 status，直接使用
  if (res.status && typeof res.status === 'function') {
    return res.status(statusCode).json(data);
  }

  // fallback 手動關閉流
  res.writeHead(statusCode, { 'Content-Type': 'application/json' });
  res.end(JSON.stringify(data));
  return;  // 完成 scope 退出以避免二次衝突
}
```

**目前驗證方式：**
```bash
# 在本地環境執行測試腳本確認 API Server 正常
cd taiwan-landlord-vietnam-tenant-system
node test-api.js   # 實際即可驗證：.status 應具備並能正常回應

# 若在 Zeabur 環境驗證無效：
# 1. 在 Zeabur Shell 中重新啟動 API Server 並確保使用同一份 simple-api-fixed.js
# 2. 觀察 Runtime Logs 是否顯示 API Server 運作在 HTTPS（8080 或其他確認 PORT）
# 3. 在程式碼中確認簡單的路由定義與邏輯（例如 /properties、/rooms 是回傳正確 JSON）
```

#### 問題 B：前端看不到後端資料

**症狀：**
- 頁面顯示「總房間數：0 間」
- 重新整理頁面後資料仍然不見

**可能原因：**
- 前端 API_URL 設定不正確或環境變數未生效
- 後端服務未啟動（502 或管理員未登入）

**解決步驟：**

**步驟 1：確認環境變數設定**
1. Zeabur Dashboard → 左側選單 →「Environment Variables」
2. 檢查與確保有：`DATABASE_URL`、`NEXT_PUBLIC_API_URL`
3. 預期的一組（如需調整可參考）：`NEXT_PUBLIC_API_URL=https://taiwan-landlord-test.zeabur.app/api`

**步驟 2：檢查後端服務狀態**
1. Zeabur Dashboard → 左側選單 →「Service」（選「以服務為單位」檢視，可能顯示 service-69ae8f...）
2. 點擊服務 → 上方選單 →「Logs」→ 確認是否有 http://localhost:8080（或實際 log）與 API 路由
3. 若無 runtime 日誌，在「Shell」中手動啟動（按開發步驟）
4. 使用 `curl` 驗收 API Server： `curl -s https://taiwan-landlord-test.zeabur.app/api/properties` 應返回 JSON

**步驟 3：清除瀏覽器快取並重新登入**
1. 瀏覽器 F12 → Application → Clear site data（可選，加強驗證）
2. 重新開啟 Vercel 網址：https://taiwan-landlord-vietnam-tenant-syst.vercel.app
3. 觀察 Network 分頁是否能看到 `/api/properties` 的成功回應

**步驟 4：調整連線模式（如必要）**
若您希望本地開發完成後完全不需依賴 Zeabur，可改用「本地後端」：
1. 在本地建立 `.env.local`，設定 `NEXT_PUBLIC_API_URL=http://localhost:3001/api`

## 🚀 快速修補參考（給開發用的記憶手冊片段）

### 後端 if 的修補重點（如再遇 same problem）
```javascript
// lib/api.ts 或 lib/cloudConnection.ts
const API_URL = process.env.NEXT_PUBLIC_API_URL || 'https://taiwan-landlord-test.zeabur.app/api';

// 建議保留此 fallback + retry（而非只靠環境變數）
const fetchWithRetry = async ( url: string, options = {} ) => {
  const MAX_RETRIES = 3;
  for (let i = 0; i < MAX_RETRIES; i++) {
    try {
      const res = await fetch(url, options);
      if (res.ok) return res;
    } catch (e) {
      if (i === MAX_RETRIES - 1) throw e;
      await new Promise(r => setTimeout(r, 1000 * (i + 1)));
    }
  }
};
```
