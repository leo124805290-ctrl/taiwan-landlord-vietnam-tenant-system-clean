## 連線與串接

### 前端呼叫後端（生產）
- 前端預設使用 `https://taiwan-landlord-test.zeabur.app/api`
- 在 Zeabur Dashboard → Environment Variables → `NEXT_PUBLIC_API_URL`
- 設定完成後必須「觸發部署」才會生效

### 本地開發
- 後端：`cd taiwan-landlord-vietnam-tenant-system` → `node simple-api-fixed.js`
- 前端：`cd .../taiwan-landlord-vietnam-tenant-system` → `npm run dev`
- 本機 .env.local → `NEXT_PUBLIC_API_URL=http://localhost:3001/api`

## 常見問題修復（快速嘗試清單）

### 新增物業失敗 / 房間無法生成
1. 在 Zeabur Shell 中 `Ctrl+C` 停止 API server
2. `nohup node simple-api-fixed.js > api-server.log 2>&1 &`
3. 視察 `tail -f api-server.log` 確認 DB 連線與無 race condition

### 502 / Service Unavailable
1. Zeabur Dashboard → Service → 確認狀態為 Running
2. 走 Runtime Logs 確認無資料庫連線錯誤
3. 若持續重啟，立即重啟：Shell → `Ctrl+C` → 重新啟動（如步驟 2）

### 頁面看不到資料
1. 確認 Vercel 環境變數 `NEXT_PUBLIC_API_URL` 值是否符合後端
2. 確認 Zeabur Shell API server 正在運行（ps aux | grep node | grep simple-api-fixed.js）
3. 前端清除快取並重新登入 / 清 site data 後再重開 URL

## 日常檢查建議（例）

- 每週 1 次：前端檢查是否有 502/500
- 每週 1 次：Zeabur Runtime Logs 確認無 DB 連線錯誤
- 每週 1 次：Stub DB 連線（je node -e script）確認根帳號可接上
- 有重大更新/變更：手動 Pull Zeabur，再推送到 GitHub 觸發 Vercel
